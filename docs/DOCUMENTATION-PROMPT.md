# 規格文件生成 Prompt

## 🎯 使用方式

當用戶請求建立規格文件時，使用以下 prompt 模板來生成文件。

---

## 📝 標準 Prompt 模板

```
請為 [功能名稱] 功能建立規格文件，檔案名稱：Spec-[功能名稱].md

## 執行步驟

1. **分析功能**
   - 搜尋相關檔案（前端、後端、資料庫）
   - 理解功能流程和架構
   - 識別關鍵 API、服務、組件

2. **建立文件結構**
   必須包含以下章節（依功能調整，不需要的章節可省略）：
   
   - 📋 概述
     - 簡短說明功能目的
     - 核心架構圖（如適用）
   
   - 🔐 [主要功能流程]
     - 流程步驟說明
     - 關鍵程式碼片段（不超過 10 行）
     - 檔案位置標註
   
   - 📝 [次要功能]
     - 狀態說明
     - 建議實作（如適用）
   
   - 🛡️ [保護/驗證機制]
     - 說明保護邏輯
     - 驗證流程
   
   - 🔌 API 端點
     - 使用表格列出所有 API
     - 包含：端點、方法、說明、是否需要認證
   
   - 📁 檔案結構
     - 清晰的目錄樹
     - 標註關鍵檔案
   
   - ⚙️ 環境變數
     - 配置說明
     - 範例值
   
   - 🔄 [狀態/流程管理]
     - 狀態說明
     - 流程圖（如適用）
   
   - 🗄️ 資料庫結構
     - Schema 說明（如適用）
     - 關聯說明
   
   - 🔧 維護重點
     - 關鍵邏輯說明
     - 注意事項
   
   - 🐛 常見問題
     - FAQ 格式
     - 問題與解答
   
   - 📚 相關文件
     - 連結到其他相關文件

3. **撰寫原則**
   - ✅ 簡潔扼要：重點說明，避免冗長描述
   - ✅ 結構清晰：使用 emoji 標題，章節分明
   - ✅ 實用導向：包含維護重點和常見問題
   - ✅ 完整性：涵蓋前端、後端、資料庫（如適用）
   - ✅ 使用表格：整理 API、配置等資訊
   - ✅ 程式碼片段：關鍵部分即可，不超過 10 行
   - ✅ 檔案路徑：標註完整的檔案位置

4. **輸出格式**
   - 檔案名稱：`docs/Spec-[功能名稱].md`
   - 格式：Markdown
   - 語言：繁體中文
   - 標題層級：使用 # 和 ##
   - Emoji：使用標準 emoji（📋 🔐 📝 🛡️ 🔌 📁 ⚙️ 🔄 🗄️ 🔧 🐛 📚）

5. **參考範例**
   - 參考 `docs/Spec-Auth.md` 的格式和結構
   - 保持一致的風格和深度
```

---

## 🔄 快速 Prompt（簡化版）

當用戶說：「[功能名稱]功能，整理成一份文件，Spec-[名稱].md。目的是為了日後維護需要，簡潔扼要」

直接執行以下步驟：

1. 搜尋並分析相關檔案
2. 按照 `Spec-Auth.md` 的格式建立文件
3. 包含：概述、流程、API、檔案結構、環境變數、維護重點、常見問題
4. 使用 emoji 標題，繁體中文，Markdown 格式
5. 儲存到 `docs/Spec-[名稱].md`

## 📋 章節選擇指南

根據功能類型選擇合適的章節：

### 認證/授權類功能
- ✅ 📋 概述
- ✅ 🔐 登入/註冊流程
- ✅ 🛡️ 路由保護
- ✅ 🔌 API 端點
- ✅ 📁 檔案結構
- ✅ ⚙️ 環境變數
- ✅ 🔄 認證狀態管理
- ✅ 🗄️ 資料庫結構
- ✅ 🔧 維護重點
- ✅ 🐛 常見問題

### API/查詢類功能
- ✅ 📋 概述
- ✅ 🔐 查詢流程
- ✅ 🔌 API 端點
- ✅ 📁 檔案結構
- ✅ ⚙️ 環境變數
- ✅ 🔄 狀態管理
- ✅ 🔧 維護重點
- ✅ 🐛 常見問題
- ❌ 🗄️ 資料庫結構（如不涉及）

### UI/組件類功能
- ✅ 📋 概述
- ✅ 🔐 使用流程
- ✅ 📁 檔案結構
- ✅ 🔧 維護重點
- ✅ 🐛 常見問題
- ❌ 🔌 API 端點（如不涉及）
- ❌ 🗄️ 資料庫結構（如不涉及）

## ✨ Prompt 變體

```
請為 [功能名稱] 功能建立詳細規格文件，檔案名稱：Spec-[功能名稱].md

重點包含：
- [重點1]
- [重點2]
- [重點3]

其他要求：
- [額外要求]
```

### 簡化版（快速建立）

```
「[功能名稱]功能，整理成一份文件，Spec-[名稱].md。目的是為了日後維護需要，簡潔扼要」
```
